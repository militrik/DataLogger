# STM32 Logic Signal Logger

Короткий приклад логера логічних сигналів для STM32 з записом на SD-карту через FatFs.  
Проєкт орієнтований на плату NUCLEO-F303RE, але може бути адаптований до інших STM32.

## Опис

Логер опитує 8 цифрових входів, що розташовані на одному GPIO-порті, виконує програмний дебаунс і записує у файл `log.csv` тільки моменти, коли будь-який з входів змінив стан.  
Кожен запис містить час (у секундах з мілісекундами від старту) і стани всіх 8 каналів у форматі CSV.

## Особливості

- 8 логічних каналів на одному GPIO-порті (послідовні біти).
- Підтримка 5V-tolerant входів (залежить від вибраних пінів МК).
- Програмний дебаунс усіх 8 бітів ("вертикальний лічильник").
- Запис тільки на фронтах/спадах (edge-based logging).
- Збереження логів на SD-карту у форматі CSV через FatFs.
- Проста інтеграція у наявний проєкт STM32CubeMX/HAL.

## Апаратні вимоги

- STM32 (приклад – NUCLEO-F303RE).
- SD-карта, підключена по SPI (реалізація `USER_diskio.c` для FatFs уже передбачається).
- 8 цифрових входів, підключених до одного GPIO-порту (послідовні біти, наприклад PA0–PA7).
- Зовнішні формувачі логічних рівнів/RC-ланцюги за потреби.

## ПЗ та структура коду

Основні файли:
- `logger.c` – реалізація алгоритму логування:
  - `Logger_Init()` – монтування FatFs, відкриття/створення `log.csv`, запис заголовка.
  - `Logger_Task()` – періодичне опитування входів, дебаунс та запис змін.
- `main.c` – виклик `Logger_Init()` після ініціалізації периферії та регулярний виклик `Logger_Task()` у головному циклі.

CubeMX/STM32 HAL:
- FatFs увімкнено з користувацьким `USER_diskio.c` (SD по SPI).
- Всі 8 GPIO налаштовані як `Input` без внутрішніх pull-up/pull-down (залежить від схеми).
- SysTick/`HAL_GetTick()` використовується для вимірювання часу (мс від старту).
